# Implementation Plan: rebuild westenb.org against Design Review v2

**Created**: 2026-04-29T16:25:15Z
**Status**: Draft
**Repository**: westenb.org
**Base commit**: `a38d781` on `main`

## Overview

Rebuild the site from scratch on a fresh branch `tufte-rebuild` off `main` to match Claude Design's "Design review v2" spec. All Markdown post content is preserved verbatim. The CSS in `_layouts/default.html` is replaced wholesale (~470 lines out, ~400 lines of design-token-driven CSS in). Three layout modes are introduced: full-width broadsheet (home), 55%-column prose (posts), dense 3-col grid (archive/tags). ETBook + JetBrains Mono are self-hosted (cherry-picked from the prior `tufte-redesign` attempt). The canonical sidenote markup pattern is adopted via a plugin rewrite. `tufte.css` itself is **not** loaded — its primitives (paper color, ETBook, sidenote pattern, ink-only links, "newthought" small-caps) are reimplemented in our own CSS, per the handoff doc's top-level recommendation.

## What we're NOT doing

- URL flattening — `permalink: /:year/:month/:day/:title/` stays. Design v2's recommendation to drop dates from URLs is explicitly out-of-scope per user.
- Per-project landing pages — `projects.md` stays as a 6-link list. Design's "Coati, Criterion DB, NIL model — own page each" is deferred.
- Email link or signup form — no service, no form, no subscriber pipeline. Drop the "Email" label from chrome. (RSS stays — `jekyll-feed` already auto-generates `/feed.xml`.)
- Print stylesheet — design line 502 mentions clean archive print rendering; deferred.
- Publishing the 3 untracked posts — `criterion-closet-by-the-numbers`, `code-driven-prompt-engineering`, `winning-is-noise` stay untracked. They have their own `.post-{slug}` scoping already; the layout will support them when shipped.
- Refactoring in-post accent callouts to ink-only — pragmatic resolution: harmonize `#c23b22` → `#a8332b` in the Koa Peat post's inline style block. (Design v2 §05 recommends ink-only in-body; user accepts pragmatic.)
- Cherry-picking layout commits from `tufte-redesign` — only the binary asset directories (et-book, jetbrains-mono) and the `sidenote.rb` rewrite are reused. The `tufte-redesign` branch's `default.html` and page rewrites are reference, not source.

## Phase 2: Sidenote plugin migration

### Tasks

- [x] Replace `_plugins/sidenote.rb` with the canonical-markup version (cherry-pick from `tufte-redesign:_plugins/sidenote.rb`):
  - `{% sidenote N body %}` emits: `<label for="sn-{rand}" class="margin-toggle sidenote-number"></label><input type="checkbox" id="sn-{rand}" class="margin-toggle"/><span class="sidenote">body</span>`
  - `{% marginnote body %}` emits: `<label for="mn-{rand}" class="margin-toggle">⊕</label><input type="checkbox" id="mn-{rand}" class="margin-toggle"/><span class="marginnote">body</span>`
  - Use `SecureRandom.hex(4)` for ids so multi-page builds don't collide
  - Existing call sites `{% sidenote 1 text %}` continue to work unchanged (the `1` is parsed but ignored)
- [x] Add the sidenote CSS block to `_layouts/default.html`:
  - `.margin-toggle { display: none }`
  - `.sidenote-number { counter-increment: sidenote-counter }` with `:after { content: counter(sidenote-counter); font-family: var(--mono); font-size: 0.7rem; vertical-align: super; color: var(--accent); margin-left: 0.1rem }`
  - `.sidenote, .marginnote { float: right; clear: right; margin-right: -60%; width: 50%; font-size: 0.95rem; line-height: 1.4; color: var(--ink-soft) }`
  - `.sidenote:before { content: counter(sidenote-counter) ". "; font-family: var(--mono); color: var(--accent) }`
  - `@media (max-width: 760px) { .sidenote, .marginnote { display: none } .margin-toggle:checked + .sidenote, .margin-toggle:checked + .marginnote { display: block; float: none; width: auto; margin: 1rem 0; padding: 0.5rem 0; border-top: 1px dotted var(--rule); border-bottom: 1px dotted var(--rule) } .sidenote-number { color: var(--accent); cursor: pointer } }`

## Risk assessment

- **Risk: Koa Peat's inline `<style>` block contains rules that conflict with the new global theme even after `.post-koa-peat` scoping.** Mitigation: review every `.post-koa-peat .x` rule in Phase 3 and verify the corresponding global rule isn't more specific. Fall back: increase specificity by adding `body .post-koa-peat .x` if needed, or strip redundant rules from the post.
- **Risk: SecureRandom-generated sidenote ids change on every build, breaking deep-link anchors.** Mitigation: per-build is fine because nothing currently links to `#sn-{id}`; ids are only used for the `<label for>`/`<input id>` pairing within a single page. If deep-linking is wanted later, swap to a deterministic id from `page.slug` + counter.
- **Risk: client-side archive search is broken in older browsers.** Mitigation: code uses only `querySelectorAll`, `addEventListener`, `dataset`, `classList` — all supported in any browser from the last 12 years. No polyfill needed.
- **Risk: visual regressions on the 3 untracked unpublished posts when they ship later.** Mitigation: their `<style>` blocks are already prefixed with `.post-{slug}` — same convention as Koa Peat post-Phase-3. Layout will support them when their files become tracked.
- **Risk: deleting `assets/css/tufte.css` breaks something undocumented.** Mitigation: file is verified not loaded via `grep -r "tufte.css" .` before deletion. The handoff doc and CLAUDE.md both confirm it's reference-only on `main`.
- **Risk: `bundle install` fails due to GitHub Pages Ruby version mismatch.** Mitigation: project uses Ruby 4.0 via Homebrew (`/opt/homebrew/opt/ruby/bin`), confirmed in handoff doc. CI builds with Ruby 3.2 per `.github/workflows/jekyll-gh-pages.yml`. If a gem upgrade introduces a Ruby version constraint, pin to a working version in `Gemfile`.

## Deviations

### Phase 2
- **Expected**: only the sidenote plugin and sidenote CSS land in Phase 2; column width comes in Phase 3.
- **Actual**: also added `.post-content, .page-content { max-width: 38rem }` in Phase 2.
- **Reason**: tufte's `float: right; margin-right: -60%` sidenote pattern requires a constrained-width column to float OUT of. Without the column rule, `<p>` was full body width (~1103px at 1199px viewport) and sidenotes pushed offscreen at x=1245. Adding the column rule was the minimum change needed to verify Phase 2's success criterion ("sidenotes render in the right gutter on desktop"). Phase 3 already plans the same `.post-content { max-width: 38rem }` rule, so this is a pull-forward, not a divergence — Phase 3's task list still applies but that one rule is already there.

### Phase 5
- **Expected**: archive page includes a client-side title-search input and ~30 lines of vanilla JS to filter rows.
- **Actual**: dropped the search entirely. Archive renders the dense list only.
- **Reason**: user explicitly opted out — "We don't need the search, don't have that many posts yet." Reasonable: 8 posts fit on a single page-down. The empty-state `.empty-msg` element and `.dense-filter` CSS were also removed.
